What is Kntnt CSS Framework

Kntnt CSS Framework (KCF) is a comprehensive set of stylsheets that together make it easy to build a website relatively quickly by chiefly overriding default values and using classes.

The framework can be used with Oxygen Builder for WordPress. But there is no dependency on either tool, so it works perfectly well with other tools or for hand written code

Different

There are many CSS frameworks and similar solutions out there. The arguably most known is Bootstrap. It's a fantastic toolkit "featuring Sass variables and mixins, responsive grid system, extensive prebuilt components, and powerful JavaScript plugins."

For better or worse, KCF is far from being as complete and competent as Bootstrap. KCF is focusing on providing a foundation with sane defaults that can be overridden with pure CSS and useful classes that can be used to accomplish everyday tasks..

How to use it

The document root must have following content:

<body>
  <header>
    <div>
      <div>…</div>
    </div>
    <div>
      <div>…</div>
    </div>
  </header>
  ⋮
  <main>…</main>
  ⋮
  <footer>
    <div>
      <div>…</div>
    </div>
    <div>
      <div>…</div>
    </div>
  </footer>
</body>

In <header>…</header>, the first … is the topbar's content (e.g. with shortcut links to essential pages), and the second … is the content of the actual header (w.g. with logo and navigation).

In <footer>…</footer>, the first … is the actual footer's content (e.g. with sitemap and social media icons), and the second … is the content of the bottombar (e.g. with copyright notice and legal disclaimers).

Replace … with nothing (i.e. <div></div>) to disable a row in in <header>…</header> or <footer>…</footer>.

The ⋮ before and after <main>…</main> represents content that is not unique for the page. Typically, the first ⋮ is used for a important message shown on all pages, while the last ⋮ is used for gallery with related articles and similar. Each ⋮ is optional. If present, it must consists of at least one pair of outer and inner wrapper:

  <section>
    <div>…</div>
  </section>

The outer wrapper, i.e. <section>…</section> above, should be a semantic element (i.e. <section>, <aside>, <article> and <nav>). If no semantic element can be used in inteded way, use <div> instead. Replace … with content.

The … between <main> and </main> represents content that is unique for the page. Typically, this is where the main content of the page, e.g. a blogpost or article, goes. It must consists of at least one pair of outer and inner wrapper described above.

Article

For a page like this, with an article, blog post or similar, <main>…</main> should contain one <article><div>…</div></article> where … is the content of the article.

<article>
  <div>
    …    
  </div>
</article>

Headline

The headline is wrapped by <h1>…</h1>.

<h1>
  Kntnt CSS Framework
</h1>

You can add a kicker or standfirst by wrapping it in <span class="kicker">…</span>and insert it in <h1>…</h1> before the headline with white a space or linefeed in-between.

<h1>
  <span class="kicker">For developers:</span>
  Kntnt CSS Framework
</h1>

You can add a deck or subhead by wrapping it in <span class="deck">…</span>and insert it in <h1>…</h1> after the headline with white a space or linefeed in-between.

<h1>
  Kntnt CSS Framework
  <span class="deck">— A foundation for modern web sites</span>
</h1>

Although not recommended, you can add both kicker and deck to a hedaline.

<h1>
  <span class="kicker">For developers:</span>
  Kntnt CSS Framework
  <span class="deck">— A foundation for modern web sites</span>
</h1>

Lead

Use the class lead on the lead paragraph to make it stand out.

<p class="lead">
  This is the lead paragraph. It should summarize the content of the article without revealing too much. The purpose is to give the reader enough so she can decide if the article is relevant, and to entice her to read on
</p>

The byline

The byline consists of a <div class="byline">…</div> wrapping one or more of following parts in order of your choice:

The author wrapped by either <a>…</a> or <span>…</span>, depending on whether the name should be a link or not.

<a href="https://github.com/TBarregren">Thomas Barregren</a>

The date when the article was published, wrapped in by <time>…</time>.

<time datetime="2021-03-28">March 28, 2021</time>

A word categorizing the article, wrapped by either <a>…</a> or <span>…</span>, depending on whether the category should be a link or not.

<a href="/news">News</a>

The result should lookk something like this:

<div class="byline">
  <time datetime="2021-03-28">March 28, 2021</time>
  <a href="https://github.com/TBarregren">Thomas Barregren</a>
  <a href="/news">News</a>
</div>

Notice that there is no separation characters (e.g. | or •) between the elements. Separator between the elements are inserted by CSS.
