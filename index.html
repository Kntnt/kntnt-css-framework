What is Kntnt CSS Framework

Kntnt CSS Framework (KCF) is a comprehensive set of stylesheets that make it easy to build a website relatively quickly by chiefly overriding default values and using classes.

The framework can be used with Oxygen Builder for WordPress. But there is no dependency on either tool, so it works perfectly well with other tools or for handwritten code.

Background

There are many CSS frameworks out there. The arguably most known is Bootstrap – an amazing toolkit “featuring Sass variables and mixins, responsive grid system, extensive prebuilt components, and powerful JavaScript plugins.” Two other full-featured CSS frameworks are Foundation and Materialize. Two leaner frameworks worth mentioning are Bulma and Pure.css. They are all great.

But they all focus on grids, buttons and forms – the stuff needed to build an excellent user interface. That’s not what we need at Kntnt.

Content – not grids

    <div>…</div>
  </section>

The outer wrapper, i.e. <section>…</section> above, should be a semantic element (i.e. <section>, <aside>, <article> and <nav>). If no semantic element can be used in inteded way, use <div> instead. Replace … with content.

The … between <main> and </main> represents content that is unique for the page. Typically, this is where the page's main content, e.g. a blog post or article, goes. It must consist of at least one pair of outer and inner wrapper described above.

Headings

Headings are used to divide a text into sections and explain what the section is about. In academic papers, technical reports, textbooks and similar, it can be beneficial to subdivide sections into subsections and further subdivide the subsections into sub-sub sections and so forth. The design of headings usually adapts to the hierarchy thus created. Usually, the headline has a visual hierarchy, where the heading of a subsection is visually "less" than its section's heading. To that end, HTML supports headings on six levels. These are <h1>…</h1>, <h2>…</h2> and so on to <h6>…</h6>.

Don’t use <h1>

Don’t use <h1>…</h1> within your text. It should only be used in the headline described above.

Shun academic headings

Academic texts, reports and many books use intermediate headings at different levels. They are divided into chapters. Each chapter is divided into larger masses of text which in turn are divided into smaller ones which may be divided into even smaller ones, and so on down to paragraphs. Above each division (except for paragraphs) is a heading whose design (for example, size) reflects the division's place in an imaginary hierarchy. These headings are written to summarize everything that falls under them.

Compare that with how newspapers and magazines use headlines. Usually, they only have subheadings, and they are used less as a summary of that particular subsection but more as a lead into what comes next.

I do not know if the different ways have any recognized names, but for the sake of reasoning, we can call them academic headings and journalistic headings, respectively. Of the two, I strongly encourage you to use journalistic headings on the web even when the text has an inherent hierarchical structure.

Subheading

A subheading is created by wrapping the heading with <h2>…</h2>. In most cases, this is the only heading you will use.

Sub-subheading

Despite my general advice, there are situations where it is beneficial for the reader to use sub-subheadings. So, therefore, KCF supports both subheadings and sub-subheadings. The last three headings were sub-subheadings.

A sub-subheading is created by wrapping the heading with <h3>…</h3>.

In KCF, <h4>…</h4> through <h6>…</h6> also represents headings, but outside of the hierarchy established by <h1>…</h1> through <h3>…</h3>. These act as headings for the particual paragraph (or possible few paragraphs) that follow.

Unordered lists

An unordered list is a list of items without implied order.

Following is an example of how to create an unordered list of Swedish kings that have ruled during the 20th century:

<ul>
  <li>Oscar II</li>
  <li>Gustav V</li>
  <li>Gustav VI Adolf</li>
  <li>Carl XVI Gustaf</li>
</ul>

The result looks like this:

  • Oscar II
  • Gustav V
  • Gustav VI Adolf
  • Carl XVI Gustaf

The character before each item can be changed by adding a class either to the <ul>-element or to an individual <li>-element. Following classes are available:

  • none
  • disc
  • circle
  • square

Inline quotes

A quote inside a text is called an inline quote. A such begins and ends with quotation marks. Although you can enter them yourself, it is higly recommeded to let the web browser do it for you. Most people don’t know how to produce typographic correct quotation marks on their keyboards, and end up using the inch-symbol "like this" instead of proper quotation marks “like this”. Modern web browsers insert correct quotation marks for tha language set with the lang attribute. What’s correct quotation marks varies from language to language. Some examples:

  • Ceci est une citation française
  • Dies ist ein deutsches Zitat
  • To jest polski cytat
  • Detta är ett svenskt citat
  • This is an English quote

To allow the web browser to insert quotation marks for you, wrap the quote with <q>…</q>. Don’t write quotation marks yourself. This is an example of how to do it:

<p>When Dave asks HAL to open the pod bay door, HAL answers: <q>I’m sorry, Dave. I’m afraid I can’t do that.</q></p>

The result looks like this:

When Dave asks HAL to open the pod bay door, HAL answers: I’m sorry, Dave. I’m afraid I can’t do that.

Another advantage of using <q>…</q> is that you can add a machine-readable link to the source with the cite attribute. Here is an example:

<p>When Dave asks HAL to open the pod bay door, HAL answers: <q cite="https://www.imdb.com/title/tt0062622/quotes/qt0396921"> I’m sorry, Dave. I’m afraid I can’t do that.</q></p>

The URL is not rendered:

When Dave asks HAL to open the pod bay door, HAL answers: I'm sorry, Dave. I'm afraid I can't do that.

Don't use <q>…</q> for other purposes than quotes. For sarcasm and similar applications of quotation marks, you must write the correct quotation marks yourself.

Blockquotes

Longer quotes that are reproduced in a separate paragraph are called block quotes. A such is wrappen in <blockquote>…</blockquote> as in this example:

<blockquote cite="https://www.imdb.com/title/tt0109830/">
  <p>Mamma said stupid is as stupid does!</p>
</blockquote>

The result looks like this:

Mamma said stupid is as stupid does!

Notice that the cite attribute isn’t rendered. It’s optional. If included, it must be a URL that designates a source document or message for the information quoted. This attribute is intended to point machines (e.g. search engines) to information explaining the context or the reference for the quote.

To add a human-readable cite, use following construct:

<figure>

  </blockquote>
  <figcaption>– Forest Gump in the movie <cite><a href="https://www.imdb.com/title/tt0109830/">Forest Gump</a></cite></figcaption>
</figure>

Note that <cite>…</cite> encloses the name of the work from which the quote comes. It should not contain the author’s name or anything else.

For a single paragraph quote, the wrapping <p>…</p> tags can be omitted:

<blockquote>
  Mamma said stupid is as stupid does!
</blockquote>

The result looks like this:

Editing text

Many people who write for the web want to forget what they are writing needs to be converted to HTML Unfortunately, word processors and text editors are lousy at creating semantically correct HTML code. Therefore, every web writer should know basic HTML and be prepared to use it. In this article, there are many examples of what you can do. However, the article is not a course in HTML. Nevertheless, in this section, we will get acquainted with HTML elements that are primarily there to give text semantic meaning and graphic style.

<em>, <dfn>, <cite>, <var> and <i>

Several HTML elements make the content they wrap rendered italicized. They differ in their semantic meaning. Choose wisely.

  • Use <em>…</em> to markup a word or a phrase that need emphasis. Example: This is indeed essential.

  • Use <dfn>…</dfn> to mark a word or a phrase that is defined. Example: In this context, pilot refers to…

  • Use <cite>…</cite> to markup a name of a creative work. Example: Shakespar wrote Hamlet.

  • Use <var>…</var> to markup a letter as a variable. Example: x = 2.

  • Use <i>…</i> to markup text that need to be differentiated from ormal text. Here are some examples listed in MDN Web Docs:

    • Alternative voice or mood
    • Taxonomic designations (such as the genus and species "Homo sapiens")
    • Idiomatic terms from another language (such as "et cetera"); these should include the lang attribute to identify the language
    • Technical terms
    • Transliterations
    • Thoughts (such as "She wondered, What is this writer talking about, anyway?")
    • Ship or vessel names in Western writing systems (such as "They searched the docks for the Empress of the Galaxy, the ship to which they were assigned.")
<strong> and <b>

Two HTML elements are rendering their wrapped content with boldface but with different semantic meaning:

  • Use <strong>…</strong> to markup a word or a phrase that need strong emphasis. Example: Warning: The coffee is hot.
  • Use <b>…</b> to bring attention to to the marked up word or phrase. MDN Web Docs suggest using it for cases like keywords in summary, product names in a review, or other spans of text whose typical presentation would be boldfaced (but not including any special importance).
<mark>

If you need to mark or highlight a portion of a text, wrap it with <mark>…</mark>.

<u> and <s>

The two elements <u>…</u> and <s>…</s> draw a line under and through, respectively, their wrapped content.

  • Use <u>…</u> as an unarticulated annotation of the wrapped text. Example: In this text, speling mistakes are underlned.
  • Use <s>…</s> to show that the wrapped content are no longer relevant or no longer accurate. Example: Use <s> to decorate your text.
<ins> and <del>

Use <ins>…</ins> and <del>…</del> to wrap text that has been inserted and deleted, respectively.

  • Use <ins>…</ins> to markup text that have been inserted. Example: He has got a new coat.
  • <del>…</del> to markup text that have been deleted. Example: The gave me his old ugly coat.
<sup> and <sub>

Use <sup>…</sup> and <sub>…</sub> to create superscript and subscript respectively.

  • Use <sup>…</sup> to display … as superscript. Example: The Pythagorean theorem: a2 + b2 = c2
  • Use <sub>…</sub> to display … as subscript. Example:

    Almost every writer's favorite molecule is C8H10N4O2 – also known as “caffeine.”

<pre>, <code>, <kbd> and <samp>

To represent computer code, input and output and similar, we have <code>, <kbd> and <samp>, respectively.

  • Use <code>…</code> to markup short fragment of computer code. Example: The tag <br> represents line feed.
  • Use <kbd>…</kbd> to illustrate keys on the keyboard. Example: Press ⌘ + S to save.
  • Use <samp>…</samp> to represent a sample output from a computer. Example: If you see Error on the screen, something went wrong.
<pre>

To represent longer text that must appear exactly as preformatted – as if it were written on a typewriter – use <pre>…</pre>.

<abbr>

Use <abbr>…</abbr> to wrap abbreviations and acronyms.

It would of course be silly to wrap every e.g., i.e., etc and so forth with <abbr>…</abbr>. Use it instead with the title attribute, as shown below, when introducing an abbreviation or acronyme that might be unknonw for the read.

<p><abbr title="HyperText Markup Language">HTML</abbr> is a markup language used to create the semantics and structure of a web page.</p>

The result is a discreet marking of the abbreviation, indicating that the users can hover over the abbreviation to see an explanation in a tooltip.

Positioning

KCF gives you the opportunity to influence the width and position of a block of content and thereby create pull quotes, calboxes and more.

By default, blocks are positioned between paragraphs with indentation. By applying one of the classes in the table below, their size and position will change.

The positioning classes above must only be used on following elements:

  • <blockquote>…</blockquote>
  • <ul>…<\ul>
  • <ol>…<\ol>
  • <dl>…</dl>
  • <table>…</table>
  • <pre>…</pre>
  • <img>
  • <figure>…</figure>
  • <aside>…</aside>
  • <div>…</div>

Moreover, the positioning classes above must only be used on an element that is not wrappen by any other element. Thus, following example is not valid:

<figure><img class="wide" … ></figure>

However, following is valid and will produce the expected result:

<figure class="wide"><img … ></figure>
